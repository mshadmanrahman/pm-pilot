# Session Workflow

## Session Start
1. MEMORY.md auto-loaded (project context, people, key facts)
2. Check `memory/handoffs/` for latest session handoff
3. Resume from where last session left off

## During Session
- Save corrections immediately as feedback in MEMORY.md
- Update project context when new facts emerge
- Track decisions and rationale
- Log blockers as they appear

## Session End
1. If work is in-progress: write handoff to `memory/handoffs/`
   - What was worked on
   - Decisions made (with rationale)
   - Blockers encountered
   - Concrete next steps
2. Save learnings to `memory/lessons.md`
3. Update MEMORY.md with new project/people context

## Rules
- Never lose a correction. If the user corrects you, persist it.
- Handoffs must be self-contained: a new session should understand context without asking.
- Lessons log captures mistakes to prevent repeats.
- Keep MEMORY.md concise. Move details to per-project files.
